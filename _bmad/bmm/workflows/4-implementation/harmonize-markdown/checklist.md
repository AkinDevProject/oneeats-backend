# Harmonize Markdown - Checklist

- [ ] H1 unique sans emoji
- [ ] Titres hiérarchiques H2/H3 sans emoji
- [ ] TOC ajoutée pour fichiers > 30 lignes (sauf exclusions)
- [ ] Badges uniquement dans README.md racine (build/tests) si souhaités
- [ ] Listes en `-`, numérotation standard pour listes ordonnées
- [ ] Fences de code avec langage indiqué quand identifiable
- [ ] Pas de trailing spaces ni double espaces inutiles
- [ ] Largeur raisonnable (~120 colonnes) quand possible
- [ ] Liens relatifs conservés ou corrigés
- [ ] Archive/ADR exclus du traitement
